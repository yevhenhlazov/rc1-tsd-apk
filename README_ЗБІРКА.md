# РЦ-1 ТСД — збірка APK (друк на Citizen CMP-40 по Bluetooth)

Цей застосунок = твій веб-інтерфейс РЦ-1 ТСД, загорнутий у нативний Android-додаток,
який друкує **напряму по Bluetooth порціями** (як Citizen MDemo), без RawBT.

Збірка відбувається **у хмарі GitHub Actions** — на твоєму Mac нічого ставити не треба.

---

## Крок 1. Акаунт і репозиторій GitHub
1. Зареєструйся на https://github.com (безкоштовно), якщо ще нема акаунта.
2. Натисни **New repository** → назва, напр. `rc1-tsd-apk` → **Private** → **Create**.

## Крок 2. Завантажити проєкт
1. На сторінці репозиторію → **Add file → Upload files**.
2. Перетягни **вміст папки `rc1-tsd-apk`** (файли `package.json`, `capacitor.config.json`,
   папки `www/` і `.github/`). Важливо завантажити саме вміст, щоб `package.json` був у корені репозиторію.
3. **Commit changes**.

## Крок 3. Зібрати APK
1. Вкладка **Actions** → побачиш запуск «Build Android APK» (стартує автоматично після завантаження).
2. Дочекайся зеленої галочки (~3–5 хв).
3. Зайди в завершений запуск → внизу секція **Artifacts** → завантаж **rc1-tsd-apk** (це zip з `app-debug.apk`).

> Якщо збірка впала (червоний хрестик) — відкрий лог, скопіюй помилку мені, поправимо.

## Крок 4. Встановити на Sunmi
1. Перекинь `app-debug.apk` на Sunmi (кабель / Google Drive / Telegram собі).
2. Відкрий файл → дозволь «Встановлення з невідомих джерел» → Встанови.
3. Запусти **РЦ-1 ТСД** → при першому друці дай дозвіл на **Bluetooth** (і Геолокацію, якщо спитає).

## Крок 5. Налаштування принтера (один раз)
- Емуляція: **CPCL** (або ZPL — головне, щоб збігалося з методом у застосунку).
- **Auto connection: Enabled**.
- Принтер **спарований** у Bluetooth Sunmi (PIN 0000), MAC `00:13:7B:5E:81:A0`.

## Крок 6. Друк
- У застосунку «Спосіб друку» обери відповідний до емуляції принтера
  (CPCL → «RawBT — CPCL растр»; ZPL → «RawBT — ZPL»).
  В APK байти підуть **нативно порціями**, не через RawBT.
- **Тест друку** → має вийти етикетка.

---

## Що всередині
- `www/` — веб-застосунок (генерує етикетку: текст + штрихкоди + растр).
- Транспорт у `www/index.html` (`sendRaw`): в APK → `bluetoothSerial` порціями по 1 КБ з паузами;
  у браузері → старий RawBT-шлях (для відладки UI).
- `.github/workflows/android.yml` — хмарна збірка APK.

## Якщо друк в APK не піде
Перевіримо/підкрутимо:
- розмір порції / паузу (CHUNK=1024, DELAY=30 у `sendRaw`),
- відповідність емуляції принтера й методу в застосунку,
- дозволи Bluetooth на Android 12+.
